Capabilities

What it does

🔎

Automated case search

Downloads the day's cause-list PDFs from highcourtchd.gov.in and scans every page for your case numbers and keywords.

📧

Email when listed

Sends a formatted alert to every recipient with matching pages, context, and the hearing date — straight to your inbox via Gmail.

📅

Weekend-aware dates

A Friday search rolls forward through Saturday, Sunday and Monday automatically, so nothing slips past the weekend.

⚙

Queued & resilient

An async task queue processes requests sequentially with retry & exponential backoff and built-in rate limiting.

📑

Full case details

Pulls case status, listing history, related cases, judgments, and impugned orders straight from the PHHC JSON API.

🔒

Secured & Dockerised

Every endpoint is gated by a bearer token, and the whole thing ships as a Docker container running as a non-root user.

Architecture

How it works

1

Queue the request

Hit POST /search/cause-list with search terms, a date, and recipient emails. It lands on an async queue.

2

Fetch the lists

The scraper pulls cause-list PDFs from the court site and, in parallel, fetches case details from the PHHC JSON API.

3

Search the PDFs

Every downloaded PDF is scanned for your terms, capturing page numbers and surrounding context for each hit.

4

Email the results

A formatted notification — matches, dates, case details — is sent to all recipients with automatic retries.

API Reference

Talking to the API

POST  /search/cause-list

Queue a cause-list search. Pass your case terms, the date to check, and who to notify. The request enters an async queue and is processed with retry logic.

Request body

  • search_terms — array of case numbers or keywords
  • date — DD/MM/YYYY to check
  • recipient_emails — who gets the results
  • case_details — optional PHHC case lookup fields
curl 
# queue a cause-list search
curl -X POST http://localhost:3080/search/cause-list \
  -H "Authorization: Bearer $AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "search_terms": ["CWP-12345-2023"],
    "date": "15/12/2024",
    "recipient_emails": ["lawyer@example.com"],
    "case_details": {
      "no": "12345",
      "type": "CWP",
      "year": "2023"
    }
  }'

GET  /health

Simple health check to confirm the service is running. No authentication required. Returns a JSON status object.

curl 
# health check
curl http://localhost:3080/health

Get Started

Running in under a minute

Clone the repo

git clone https://github.com/karantrehan3/cause-list-checker.git

2
Configure secrets

Copy .env.example to .env and fill in your Gmail app password and auth token.

3
Start with Docker

docker compose up --build serves on port 3080.

4
Queue a search

Hit POST /search/cause-list with your terms and wait for the email.
