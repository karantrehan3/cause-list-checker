Cause List Checker

Searches Punjab & Haryana High Court cause-list PDFs for your cases and emails you the moment one is listed for a hearing.

View on GitHub How it works

FastAPI · Dockerised · Gmail notifications · queued & weekend-aware

What it does

🔎

Automated case search

Downloads the day's cause-list PDFs from highcourtchd.gov.in and scans every page for your case numbers and keywords.

📧

Emailed when listed

Sends a formatted alert to every recipient with the matching pages, context, and the hearing date — straight to your inbox via Gmail.

📅

Weekend-aware dates

A Friday search rolls forward through Saturday, Sunday and Monday automatically, so nothing slips past the weekend.

⚙️

Queued & resilient

An async task queue processes requests sequentially with retry & backoff and built-in rate limiting, so the court site is never hammered.

📑

Full case details

Pulls case status, listing history, related cases, judgments, copy petitions and impugned orders straight from the PHHC JSON API.

🔐

Secured & Dockerised

Every endpoint is gated by a bearer AUTH_TOKEN, and the whole thing ships as a Docker container that runs as a non-root user.

Under the hood

FastAPI server

RESTful endpoints for case-search operations, with token-based authentication middleware.

Queue manager

Asynchronous task queue with retry logic, unique task IDs, and live status monitoring.

PDF searcher

Scans downloaded cause-list PDFs for your terms, returning page numbers and surrounding context.

Web scraper

Downloads the lists from highcourtchd.gov.in and fetches case details via the PHHC JSON API.

Email service

Sends formatted result notifications over Gmail SMTP, using an app password for auth.

Auth layer

Bearer-token authentication on every route keeps the API private to authorised callers.

Talking to the API

Queue a search with your case terms, the date to check, and who to notify — then a health check to confirm the service is up. 

# is it up?
curl http://localhost:3080/health

# queue a cause-list search
curl -X POST http://localhost:3080/search/cause-list \
  -H "Authorization: Bearer $AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "search_terms": ["CWP-12345-2023"],
    "date": "15/12/2024",
    "recipient_emails": ["lawyer@example.com"],
    "case_details": { "no": "12345", "type": "CWP", "year": "2023" }
  }'

How it works

Queue the request. Hit POST /search/cause-list with your search terms, a date, and the recipient emails. It lands on an asynchronous queue.

Fetch the lists. The scraper pulls the cause-list PDFs for each date and, in parallel, fetches detailed case status from the PHHC JSON API.

Search the PDFs. Every downloaded PDF is scanned for your terms, capturing the page number and surrounding context for each hit.

Email the results. A formatted notification — matches, dates, case details — is sent to all recipients, with automatic retries on failure.
